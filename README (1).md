## 🧠 ภารกิจที่ 1: Training Strategy

### 1. จะใช้ประเภทที่ดินกี่ Class?

งานนี้ใช้ **7 classes** ตามข้อมูลฝึกที่มีอยู่แล้วในพื้นที่นครนายก ได้แก่

1. **Bareland** — พื้นที่โล่ง/ดินเปลือย/ไม่มีพืชปกคลุมเด่นชัด  
2. **Buildup** — พื้นที่สิ่งปลูกสร้าง ถนน ชุมชน และโครงสร้างพื้นฐาน  
3. **Fieldcrop** — พื้นที่พืชไร่หรือพื้นที่เกษตรเชิงแปลง  
4. **Forest** — พื้นที่ป่าไม้และพืชพรรณหนาแน่น  
5. **Orchard** — สวนผลไม้/สวนที่มีรูปแบบแปลงไม้ผล  
6. **Perennial** — พืชยืนต้นที่ไม่ใช่สวนผลไม้เด่นชัด หรือแปลงไม้ยืนต้นผสม  
7. **Water** — แหล่งน้ำ เช่น คลอง หนอง บึง อ่างเก็บน้ำ

> แม้โจทย์จะแนะนำ 4–6 classes แต่ในกรณีนี้เลือก 7 classes เพราะสอดคล้องกับโครงสร้างข้อมูลฝึกเดิมของพื้นที่นครนายกและสะท้อน landscape ของพื้นที่ได้ละเอียดกว่า  
> อย่างไรก็ตาม หากพบว่า `Orchard` และ `Perennial` สับสนกันมากในผลลัพธ์ สามารถพิจารณา merge class ในงานเวอร์ชันถัดไปได้

### 2. จะหา Training Samples อย่างไร?

ใช้ **training polygons ที่เตรียมไว้ใน `Trainning.shp`** เป็นหลัก และถือเป็น **manual / locally adapted training data** มากกว่าการคัดลอกตัวอย่างจาก tutorial ตรง ๆ  
แนวคิดนี้สอดคล้องกับโจทย์ที่ต้องการให้ training data ผูกกับพื้นที่จริง

หลักการคัดเลือก training data ที่เหมาะสม:
- กระจายตัวอย่างให้ครอบคลุมหลายส่วนของจังหวัด
- พยายามครอบคลุมความแปรผันภายใน class เดียวกัน
- หลีกเลี่ยง polygon ที่ทับซ้อนหลายการใช้ที่ดิน
- ตรวจสอบให้สอดคล้องกับภาพ raster จริงก่อนนำไปฝึก

### 3. จะแบ่ง Train / Validation อย่างไร?

งานนี้ใช้ **stratified 80/20 split ในระดับ polygon** ไม่ใช่สุ่มแยกในระดับ pixel จาก polygon เดียวกัน  
เหตุผลคือการสุ่มแบบ pixel-level มีโอกาสเกิด **data leakage** และทำให้ accuracy สูงเกินจริง

#### 80/20 random split
ข้อดี:
- ทำง่าย
- รันเร็ว
- เหมาะกับงานทดลองเบื้องต้น

ข้อจำกัด:
- ถ้าสุ่มในระดับ pixel อาจทำให้ train และ validation อยู่ใกล้กันมากเกินไป
- accuracy ที่รายงานอาจ optimistic

#### Spatial cross-validation
ข้อดี:
- เข้มงวดกว่า
- วัดความสามารถในการ generalize ไปยังพื้นที่ใหม่ได้ดีกว่า

ข้อจำกัด:
- ทำยากกว่า
- ต้องมีจำนวน sample ที่เพียงพอและกระจายเชิงพื้นที่ดี

**ข้อสรุป:**  
ใน notebook นี้เลือกวิธี **polygon-level stratified split** เป็นจุดสมดุลระหว่างความถูกต้องทางวิธีวิทยาและความง่ายในการรันบน Colab  
ถ้าต้องการประเมินเข้มงวดขึ้นในงานต่อยอด ควรเพิ่ม spatial block cross-validation

### 4. Feature ที่จะใช้มีอะไรบ้าง? เพราะอะไร?

ใช้ 2 ชุด feature เพื่อเปรียบเทียบ:

#### A. Bands only
- Blue
- Green
- Red
- NIR

เหตุผล:
- เป็น spectral information พื้นฐานจาก Sentinel-2
- ใช้แยก vegetation / non-vegetation / water ได้ดีในระดับหนึ่ง
- สอดคล้องกับเอกสารอ้างอิงเดิมของพื้นที่

#### B. Bands + Spectral Indices
- Blue
- Green
- Red
- NIR
- NDVI
- GNDVI
- NDWI
- SAVI
- EVI

เหตุผล:
- **NDVI / GNDVI** ช่วยแยกความหนาแน่นและความแตกต่างของพืชพรรณ
- **NDWI** ช่วยเน้นน้ำ
- **SAVI** ช่วยลดอิทธิพลของดินในพื้นที่เกษตรหรือดินโล่ง
- **EVI** ช่วยแยกความแตกต่างของพืชในสภาพที่ canopy หนาแน่นขึ้น

## 🌟 ผลอ้างอิงจากรายงานเดิมของพื้นที่นครนายก

จากรายงานอ้างอิงที่แนบมา ผลการจำแนกของพื้นที่นครนายกในงานก่อนหน้าเป็นดังนี้

| วิธี | OA โดยประมาณ | Kappa โดยประมาณ | ข้อสังเกต |
|---|---:|---:|---|
| Random Forest | ~82% | ~0.77 | ดีที่สุด |
| Maximum Likelihood | ~68% | ~0.60 | รองลงมา |
| ISODATA | ~62% | ~0.54 | ต่ำสุด |

### การตีความเบื้องต้น
- Random Forest เหมาะมากกับข้อมูลหลาย class และความสัมพันธ์ non-linear
- วิธีแบบดั้งเดิมอย่าง Maximum Likelihood ยังพอใช้ได้ แต่ด้อยกว่า RF
- Unsupervised clustering แบบ ISODATA ให้ baseline ที่ใช้เทียบได้ แต่ไม่ใช่วิธีที่แม่นที่สุดสำหรับโจทย์นี้

## 🌿 ภารกิจที่ 3: วิเคราะห์ Feature Importance

### คาดว่า feature ใดสำคัญที่สุด?
ในบริบทของ Sentinel-2 ที่ใช้ Blue/Green/Red/NIR:
- **NIR** มักสำคัญมากในการแยกพืชกับไม่ใช่พืช
- **NDVI / GNDVI** มักสำคัญกับการแยกความแตกต่างระหว่างป่า พืชไร่ และพืชยืนต้น
- **NDWI** มักสำคัญในการแยกน้ำออกจากพื้นผิวอื่น
- **Blue** อาจมีความสำคัญต่ำกว่า feature อื่นในหลายกรณี เพราะมักซ้ำซ้อนกับข้อมูลที่ดึงได้จาก index

### สอดคล้องกับความเป็นจริงทางกายภาพหรือไม่?
สอดคล้อง เพราะ:
- พืชสะท้อนในช่วง NIR สูง
- น้ำมีลักษณะ spectral signature ต่างจากพื้นผิวแห้งอย่างชัดเจน
- พื้นที่ป่า สวน และพืชยืนต้นบางชนิดอาจทับซ้อนกันใน visible bands แต่แยกได้ดีขึ้นเมื่อเพิ่ม vegetation indices

### ถ้าตัด feature สำคัญต่ำออก ความแม่นยำจะเปลี่ยนอย่างไร?
โดยทั่วไปมี 2 กรณี
1. **ลดลงเล็กน้อยหรือแทบไม่เปลี่ยน**  
   แปลว่า feature ที่ตัดออกมีความซ้ำซ้อนสูง
2. **ลดลงชัดเจน**  
   แปลว่า feature เหล่านั้นช่วยแยก class ที่สับสนกันอยู่จริง

> Notebook นี้มี cell สำหรับทดลองตัด feature ที่อยู่ในกลุ่มความสำคัญต่ำออก แล้ว retrain เพื่อตอบคำถามนี้ด้วยข้อมูลจริง

## ❓ ภารกิจที่ 4: ประเมินความไม่แน่นอน

### 1. Class ใดถูกสับสนกันมากที่สุด?
จาก confusion matrix ในรายงานอ้างอิงเดิม พบว่ากลุ่มที่มีโอกาสสับสนกันคือ
- **Perennial ↔ Fieldcrop**
- **Orchard ↔ Fieldcrop**
- **Buildup ↔ Fieldcrop** บางส่วน

### เพราะอะไรในทางกายภาพ?
- พืชไร่ สวน และพืชยืนต้นอาจมีค่า spectral ใกล้กันในบางฤดูกาล
- พื้นที่เกษตรผสมและขอบแปลงมี mixed pixels
- สิ่งปลูกสร้างในชนบทอาจปะปนกับแปลงเกษตรและดินโล่ง

### 2. บริเวณที่โมเดลไม่แน่ใจอยู่ที่ไหน?
คาดว่าจะอยู่บริเวณ:
- ขอบแปลงเกษตร
- พื้นที่เปลี่ยนผ่านระหว่างสวนกับพืชไร่
- ขอบแหล่งน้ำ
- พื้นที่ชุมชนชนบทที่ผสมสิ่งปลูกสร้างกับต้นไม้

ใน notebook มีการสร้าง:
- `max probability map`
- `low confidence mask (probability < 0.60)`

เพื่อใช้ชี้ตำแหน่งเหล่านี้จริงบนแผนที่

### 3. ถ้าต้องบอก stakeholder ว่าแผนที่เชื่อถือได้แค่ไหน จะพูดอย่างไร?
คำตอบที่เหมาะสมคือ:

> แผนที่นี้เชื่อถือได้ในระดับใช้งานวิเคราะห์ภาพรวมเชิงพื้นที่ โดยเฉพาะการแยกน้ำ ป่า และพื้นที่เกษตรหลัก  
> แต่บริเวณขอบแปลง พื้นที่เกษตรผสม และ class ที่มีลักษณะทางสเปกตรัมใกล้กันยังมีความไม่แน่นอนสูง  
> ดังนั้นควรใช้ร่วมกับข้อมูลภาคสนาม ภาพความละเอียดสูง หรือข้อมูลหลายช่วงเวลา หากต้องการตัดสินใจเชิงปฏิบัติการในระดับแปลง
